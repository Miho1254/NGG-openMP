    return parent::update($attributes, $options);

### Bước 3: Custom Whirlpool Hasher

**⚠️ BẪY #1 - WHIRLPOOL HOA/THƯỜNG (SILENT KILLER):**
- Dữ liệu trong `backup.sql` lưu hash **IN HOA** (VD: `6C3995DE505151B9...`)
- Hàm PHP `hash('whirlpool', ...)` trả về **chữ thường** (`6c3995de505151b9...`)
- SA-MP dùng `WP_Hash()` (Pawn native) luôn trả về **IN HOA**
- `strcmp()` trong SA-MP so sánh case-sensitive → nếu không match = "sai mật khẩu"
- **FIX: Bắt buộc `strtoupper(hash('whirlpool', $value))` trong hasher**

**File mới:** `SPRP-UCP/app/Hashing/WhirlpoolHasher.php`

```php
class WhirlpoolHasher implements \Illuminate\Contracts\Hashing\Hasher {
    public function make($value, array $options = []): string {
        return strtoupper(hash('whirlpool', $value));
    }
    public function check($value, $hashedValue, array $options = []): bool {
        return strtoupper(hash('whirlpool', $value)) === $hashedValue;
    }
    public function needsRehash($hashedValue, array $options = []): bool {
        return false;
    }
}
```

**LƯU Ý VỀ SALT:** SA-MP ghép salt VÀO password trước khi hash: `WP_Hash(password + salt)`.
Vì vậy hasher ở trên chỉ hash chuỗi THÔ. Việc ghép salt phải xảy ra Ở NGOÀI hasher (trong LoginForm authenticate hoặc register), truyền vào hasher chuỗi đã ghép sẵn `$password . $salt`.

**File mới:** `SPRP-UCP/app/Providers/WhirlpoolHashServiceProvider.php`

Override binding `hash` trong service container:
```php
$this->app->singleton('hash', function () {
    return new \App\Hashing\WhirlpoolHasher();
});
```

Đăng ký provider trong `config/app.php` (thêm vào mảng `providers`).

**PHP 8.x hỗ trợ:** Hàm `hash('whirlpool', ...)` có sẵn khi extension `hash` được bật (mặc định). Không cần cài thêm thư viện.

### Bước 4: Sửa Authentication Flow

**File:** `SPRP-UCP/app/Livewire/Forms/LoginForm.php`

**⚠️ BẪY #2 - XUNG ĐỘT DỮ LIỆU REAL-TIME (DATA CORRUPTION):**
- Cột `Online` trong `accounts` đánh dấu người chơi đang online trong server SA-MP
- `Online = 0` = offline, `Online = servernumber` = đang online
- Khi player connect: `UPDATE accounts SET Online=%d, LastLogin=NOW()` (`mysql.pwn:1428`)
- Khi player disconnect: `UPDATE accounts SET Online=0` (`mysql.pwn:1446`)
- **Nếu UCP ghi dữ liệu (mua đồ, đổi pass, đổi tên) khi player đang online, server SA-MP sẽ ghi đè RAM → DB khi disconnect → dữ liệu UCP bốc hơi!**

**FIX:** Trong `LoginForm::authenticate()` SAU KHI login thành công, kiểm tra `Online`:
```php
if ($user->Online != 0) {
    Auth::logout();
    throw ValidationException::withMessages([
        'form.account_name' => 'Tài khoản đang online trong game. Vui lòng thoát game trước khi đăng nhập UCP.',
    ]);
}
```

Sửa `authenticate()` hoàn toàn thủ công (KHÔNG dùng `Auth::attempt()`):
```php
public function authenticate(): void
{
    $this->ensureIsNotRateLimited();

    // Tìm user theo Username
    $user = \App\Models\User::where('Username', $this->account_name)->first();

    if (!$user) {
        RateLimiter::hit($this->throttleKey());
        throw ValidationException::withMessages([
            'form.account_name' => trans('auth.failed'),
        ]);
    }

    // Ghép password + salt rồi hash Whirlpool (IN HOA)
    $hashedInput = strtoupper(hash('whirlpool', $this->account_password . $user->Salt));

    if ($hashedInput !== $user->Key) {
        RateLimiter::hit($this->throttleKey());
        throw ValidationException::withMessages([
            'form.account_name' => trans('auth.failed'),
        ]);
    }

    // Check Online status - chặn login nếu đang trong game
    if ($user->Online != 0) {
        throw ValidationException::withMessages([
            'form.account_name' => 'Tài khoản đang online trong game. Vui lòng thoát game trước.',
        ]);
    }

    RateLimiter::clear($this->throttleKey());
    Auth::login($user, $this->remember);
}
```

**Middleware bảo vệ:** Tạo middleware `CheckPlayerOffline` để chặn các thao tác ghi dữ liệu quan trọng (mua đồ, đổi pass) nếu `Online != 0`:
```php
// Áp dụng cho các route: change-password, shop, transfer-money, ...
if (auth()->user()->Online != 0) {
    return back()->with('error', 'Vui lòng thoát game trước khi thao tác trên UCP.');
}
```

**Bẫy #1 - Whirlpool HOA/THƯỜNG (SILENT KILLER):**

- Source proof: `backup.sql` line 553 → hash bắt đầu bằng `6C3995DE` (HOA)

**Bẫy #2 - Xung đột dữ liệu Real-time (DATA CORRUPTION):**
- Cột `Online` trong `accounts` đánh dấu player đang trong game
- `Online = servernumber` = đang online, `Online = 0` = offline
- Nếu UCP ghi DB khi player đang online, SA-MP ghi đè RAM → DB khi disconnect
- **FIX: Check `Online != 0` → chặn login/thao tác UCP**
- Source proof: `mysql.pwn:1428` → `SET Online=%d`, `OnPlayerLoad.pwn:39` → `if(pOnline != 0)`

**Bẫy #3 - Không được xóa Character Model (VỠ UI):**
- 80% trang UCP inject `$user->characters->first()->player_xxx`
- Xóa model = crash toàn bộ dashboard, profile, admin records
- **FIX: Giữ model, đổi `$table = 'accounts'`, dùng Accessor map tên cột**
- Source proof: `profile.blade.php:19-21`, `characters/index.blade.php:20`, `adminrecord/index.blade.php:19`

**Bẫy #4 - UpdateDate bị bỏ rơi (CHẾT LÂM SÀNG):**
- SA-MP dùng `UpdateDate > NOW() - INTERVAL 1 WEEK` để quét account active
- Nếu UCP không update `UpdateDate`, account bị đánh dấu "inactive"
- **FIX: Override `saving()` event trong User model → tự set `UpdateDate = now()`**
- Source proof: `mysql.pwn:1957-1958` → `SELECT ... WHERE UpdateDate > NOW() - INTERVAL 1 WEEK`

### Rủi ro khác

1. **Single Character Model:** SA-MP mỗi account = 1 nhân vật. UCP được thiết kế cho multi-character. Cần disable tính năng tạo nhân vật thứ 2/3 (ẩn nút "Create Character" slot 2+).

2. **Email trống:** Nhiều account trong SA-MP có `Email = ''`. Login trên UCP cho phép dùng Username (không bắt buộc email). Register mới bắt buộc email.

3. **Bảo mật Whirlpool:** Whirlpool yếu hơn Bcrypt. Sau khi merge xong, nên xem xét migrate password sang Bcrypt dần (rehash on login).

4. **Filament Admin:** Cần `AdminLevel` đủ cao (VD >= 99999) để truy cập admin panel. Sửa `canAccessPanel()`.

5. **Bảng `gangzones`/`properties`:** Không thấy trong backup.sql. Có thể gamemode tạo runtime hoặc ở DB riêng. Cần hỏi chủ server.
